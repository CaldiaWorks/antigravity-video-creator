# 要件定義書リスト

このディレクトリでは、プロジェクトの各機能やモジュールの要件定義書を管理します。

## 採番ルール
GitHub Issue が発行される前の要件定義書には、以下のルールで独自ID（プレフィックス）を付与します。

- **形式**: `REQ-###`（例: REQ-001）
- **ファイル名**: `REQ-###-feature-name.md`
- **Issue移行後**: 要件確定後に GitHub Issue を発行し、その後の設計・実装ドキュメントは Issue 番号（例: `#123`）をプレフィックスとして使用します。

## 運用フロー
1. 新規要件が発生したら、管理表に新しい `REQ-###` を登録する。
2. `template.md` をコピーし、`REQ-###-feature-name.md` という名前で保存する。
3. 要件が確定したら GitHub Issue を作成し、管理表の「Issue」列に番号を記載する。
